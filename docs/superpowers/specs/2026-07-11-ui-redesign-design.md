# Taskit UI 重构设计

> 将分散在多个页面的功能整合到一个单页布局中，优化框架选择交互和结果展示方式。

**全局约束：默认主题为亮色，主题偏好持久化到 localStorage。**

---

## 背景

当前 Taskit 的流程是两页结构：

1. `/` — HomePage，上传模型 + 配置参数 + 创建任务
2. `/tasks/:id` — TaskPage，查看分析结果

实际上用户的使用场景是：**一个页面完成所有操作**——上传 → 配置 → 运行 → 看结果，不应该有页面跳转。

## 范围

### 1. 单页布局（合并 HomePage + TaskPage）

将两个路由合并为一个 `/` 页面，页面分上下两个区域：

```
┌──────────────────────────────────────────────┐
│  TopNav (Taskit logo + 用户 + 主题)           │
├──────────────────────────────────────────────┤
│  配置区（初始 = 上传区，有文件后 = 配置面板）    │
│                                              │
│  ┌─ 基准 ───┐ ┌─ 对比框架1 ─┐ ┌─ 对比框架2 ┐ │
│  │ ONNX RT  │ │ [TensorRT ▼]│ │ [+] 添加   │ │
│  └──────────┘ └─────────────┘ └────────────┘ │
│                                              │
│  高级参数 (折叠)                              │
│  ┌ Batch Size ────┐ ┌ 推理精度 ───────┐      │
│  │ [4           ▼]│ │ [AUTO         ▼]│      │
│  └────────────────┘ └────────────────┘      │
│  输入数据: [随机数据] [文本] [文件]            │
│                                              │
│  [🚀 开始分析]                               │
├──────────────────────────────────────────────┤
│  结果区（任务运行后显示）                       │
│                                              │
│  ┌ 配置摘要条 (可展开编辑) ────────────────┐  │
│  └────────────────────────────────────────┘  │
│                                              │
│  模型信息卡片                                 │
│  层余弦列表 + 雷达图                           │
│  全量层表格 (多列: 基准 | 对比1 | 对比2)       │
│  执行链路图 (graphData)                       │
│                                              │
│  ↓ 点击某层 → 悬浮小窗                        │
│                                              │
└──────────────────────────────────────────────┘
```

**关键行为**：
- 页面初始状态：显示拖拽上传区 + 最近任务列表
- 拖拽文件后：上传区消失，配置面板展开
- 点"开始分析"后：配置面板收缩为一行摘要条（可展开重新编辑），结果区显示运行进度
- 运行完成后：结果区展示分析结果
- 整个过程 URL 不变，无跳转

### 2. 框架选择器（左右三列布局）

将原来的纵向"基准框架 + 对比框架多选"改为横向三列：

- **默认**：基准 ONNX Runtime + 对比框架 1（下拉选 TensorRT / OpenVINO）
- **点[+]**：追加对比框架 2（选剩下的那个，最多 2 个）
- 每列有 ✕ 移除按钮
- 已选的框架不在下拉选项中重复出现
- 按钮文案动态："开始分析（TensorRT）" / "开始分析（TensorRT + OpenVINO）"

### 3. 更多推理配置折叠

Batch Size、推理精度、输入数据三项折叠在"更多推理配置"中，默认收起。`<details>` 元素实现，无需额外 JS。

### 4. 结果区 — 多列表格

当前：单框架下拉切换，每次只看一个框架的数据。

改为：表格的列按框架排列，左到右对应配置区的顺序：

```
层名 | 类型 | ONNX Runtime (基准) | TensorRT    | OpenVINO
     |      | 余弦     最大误差   | 余弦   误差  | 余弦   误差
```

- 使用 `position: sticky` 固定层名列在最左
- 每对列显示余弦相似度 + 最大误差
- 绿色/红色标记通过/失败
- 当只有 1 个对比框架时，第三列（OpenVINO）不显示

### 5. 层详情 → 悬浮小窗

当前：右侧 380px 面板，选中层时滑出。

改为：点击表格行时，在该行附近弹出浮动卡片：

- 显示输入/输出形状
- 每个框架的 4 项精度指标（余弦相似度、最大绝对误差、平均绝对误差、信噪比）
- 点击空白处或 ✕ 关闭
- 点另一行直接替换内容

### 6. URL 设计

```
/                          → 空页面 / 配置界面（主页）
/tasks/42                  → 直接打开并展示任务 #42（可分享、可书签）
```

- 主流程在 `/`，所有操作（上传 → 配置 → 运行 → 看结果）无跳转
- `/tasks/42` 渲染的是**同一个页面组件**，但初始化时通过 URL 参数预加载任务 #42 的结果
- 历史任务抽屉点击某条记录时，跳转 URL 到 `/tasks/:id`（同时更新页面内容）
- 浏览器前进/后退正常可用

### 7. Toast 通知

新增 `sonner`（shadcn/ui 默认方案）或使用 `@radix-ui/react-toast`（已在 package.json 中），在以下操作后弹出反馈：

- 任务创建成功 → "分析任务已提交"
- 任务完成 → "分析完成"
- 任务失败 → "分析失败: {错误信息}"
- 重试成功 → "任务已重新提交"

### 7. 轮询生命周期修复

`TaskForm` 中 `setInterval` 启动轮询后，如果在组件卸载时没有清理，会继续发请求报错。

修复方案：将轮询逻辑移到 `useEffect` 中，返回 cleanup 函数 `clearInterval`。

### 8. 补齐 mock 数据

`mockData.ts` 中补充 `graphData` 字段，使 `ExecutionTree` 在开发时可渲染。

## 依赖关系

| 项 | 前置依赖 | 工作量估计 |
|----|---------|-----------|
| 1. 单页布局 | 无 | ~60min |
| 2. 框架选择器 | 1 | ~30min |
| 3. 高级参数折叠 | 2 | ~10min |
| 4. 多列表格 | 1 | ~45min |
| 5. 悬浮小窗 | 1 | ~30min |
| 6. Toast 通知 | 1 | ~20min |
| 7. 轮询修复 | 1 | ~5min |
| 8. mock 数据补齐 | 无 | ~10min |

第 7、8 项可与其他项并行。其余项以第 1 项（单页布局）为基础。

- 面包屑导航
- 结果导出/报告功能
