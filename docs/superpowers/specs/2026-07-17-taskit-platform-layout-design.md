# Taskit 平台布局重构设计

> 以 DeployAgent UI 为蓝本，将 Taskit 从单工具页面重构为平台级布局，ModelDiff（精度比对）和 DeployAgent（模型部署）作为子产品统一在 Taskit 品牌下。

---

## 背景

当前 Taskit 是一个单页工具：48px TopNav + 内容区。随着产品线扩展（新增 DeployAgent 模型部署管理），需要升级为平台级布局：

1. **品牌统一**：Taskit 作为平台品牌，子产品共享同一套视觉框架
2. **布局对齐**：与 DeployAgent UI 设计稿的 Header + Sidebar + Content + Drawer 四区布局对齐
3. **交互升级**：从"页面切换"变为"表格为主视图，Drawer 处理详情和表单"

## 目标

- 建立可在 ModelDiff 和 DeployAgent 之间复用的布局骨架
- 主内容区改为任务表格（类似 DeployAgent 的大盘视图）
- 新建任务和任务详情都通过右侧 Drawer 完成，不切换页面
- 预留 DeployAgent 子产品入口（当前 disabled/"即将上线"）

### 四区职责

| 区域 | 宽度/高度 | 职责 |

| Sidebar | w-[240px] | 子产品切换（精度比对 / 部署工坊） |
| Content | flex-1 | 任务表格（子产品各自的任务列表） |
| Drawer | w-[500px] | 新建任务表单 + 任务详情（双模式） |

参考 DeployAgent header 结构：

```

│ └────────────────────┘ ┊ └──────────────┘ └────────────┘│

```

- **高度**：76px（和 DeployAgent 一致）
- **背景**：`bg-white` + `border-b border-sky-100`
- **左侧**：品牌 logo + "TASKIT" + `v2.0` 标签
- **中间**：可选的子产品状态指示器（如 ModelDiff 编译状态、后端连接状态）
- **右侧**：主操作按钮 [新建任务]，蓝色填充

现有 `TopNav.tsx` 废弃，替换为 `Header.tsx`（在 `src/core/components/` 下）。

### 交互规则

- **点击子产品** → 切换 Main Content 为该产品的任务表格
- **当前产品高亮**（`bg-sky-50` + `text-brand-accent` + 左侧蓝色竖条）
- **非活跃产品**（如部署工坊）显示为灰色，可点击但展示"即将上线"空态页（插画 + "模块开发中，敬请期待"）
- **"全部任务记录"** 是跨模块入口，展示所有子产品的任务归档（按时间倒排，每行标注所属产品标签如 `[精度比对]` / `[部署工坊]`）
- **底部区域**展示系统信息（后端节点 IP、连接状态）

> 注：如果"全部任务记录"在 MVP 中暂不需要，可省略该条目，侧边栏只保留子产品导航 + 系统信息。

### 视觉风格

- 宽度 240px，背景 `bg-white`，右侧边框 `border-r border-sky-100`
- 导航项间距 4px，圆角 `rounded-xl`，hover 时浅 sky 背景
- 子产品和通用区之间用 `h-px bg-slate-100` 分隔
- 底部系统信息用浅 sky 背景小卡片

### 表格列定义 (ModelDiff)

| 列 | 内容 | 宽度 |
|----|------|------|
| 部署任务/目标框架 | 模型名(mono) + 框架组合标签 | 28% |
| 精度指标 | 余弦相似度 + 端侧延时 | 18% |
| 状态 | READY/PENDING/FAILED/COMPILING 标签 | 16% |
| 完成时间 | 格式化时间 | 16% |
| 操作 | 查看详情/下载/审阅 按钮 | 22% |

### 状态标签体系 (复用 DeployAgent)

| 状态 | 视觉 | 含义 |
|------|------|------|
| READY | 🟢 emerald 背景 + check 图标 | 已完成，可下载部署包 |
| PENDING | 🟡 amber 背景 + warning 图标 | 精度低于阈值，需人工审阅 |
| COMPILING | 🔵 slate 背景 + 齿轮动画 | 正在执行中 |
| FAILED | 🔴 red 背景 + 错误图标 | 执行失败，可重试 |

### 过滤栏

- 状态过滤（全部/READY/PENDING/COMPILING/FAILED）
- 框架过滤（全部/ONNX Runtime/TensorRT/OpenVINO）
- 搜索框（按模型名搜索）
- 背景白色 + 圆角卡片，与 DeployAgent 一致

## 8. 状态管理

新增 `src/stores/appStore.ts`（或扩展 `uiStore.ts`）：

```ts
interface AppState {
  activeModule: 'model-diff' | 'deploy-agent'  // 当前子产品
  drawerMode: 'closed' | 'new-task' | 'task-detail'
  drawerTaskId: number | null
}
```

- `activeModule` 由 Sidebar 点击切换
- `drawerMode` + `drawerTaskId` 决定 Drawer 的内容和状态
- 切换子产品时，自动关闭 Drawer
